## QUICK REFERENCE: Red Flag Patterns

When evaluating E₁ ⇒ E₂, watch for these structural red flags:

| Red Flag | Indicates | Confidence |
|----------|-----------|------------|
| Non-commutative operation | E₁ ⇒ commutativity likely FALSE | 95% |
| Non-associative operation | E₁ ⇒ associativity likely FALSE | 87% |

**Key insight**: If the magma shows ANY red flag for the target property,
the implication is probably false. Look for counterexamples in 2-4 element magmas.

## EQUATION TAXONOMY

### Structural Classes

**Associative**: ((x·y)·z) = (x·(y·z))
- Creates semigroup structure
- Enables parentheses omission
- Common in: addition, multiplication

**Commutative**: (x·y) = (y·x)
- Order-independent operations
- Common in: addition, AND, OR, XOR

**Idempotent**: (x·x) = x
- Elements are projections
- Common in: meet, join, min, max

### Identity Properties

**Left Identity**: ∃e: (e·x) = x for all x
**Right Identity**: ∃e: (x·e) = x for all x
**Two-sided Identity**: Both hold (then e is unique)

**Theorem**: If both left and right identity exist for same e,
they define a unique identity element.

### Distributive Properties

**Left Distributive**: x·(y·z) = (x·y)·(x·z)
**Right Distributive**: (x·y)·z = (x·z)·(y·z)

**Medial**: (x·y)·(z·w) = (x·z)·(y·w)
- Stronger than distributive in semigroups
- Implies certain symmetric properties

### Left Identity ⇒ Right Identity: ✗ FALSE
**Counterexample**: Left-only identity structures
- e·x = x for all x, but x·e ≠ x
**Red flags**: One-sided identity
**Confidence**: 85%

## PROOF STRATEGY GUIDE

### Strategy 1: Trivial Check
**When**: E₁ and E₂ have identical form
**Action**: Return TRUE by reflexivity
**Example**: (x·y)·z = (x·y)·z ⇒ anything about itself

### Strategy 2: Structural Similarity
**When**: E₁ and E₂ have same structure, different variables
**Action**: Apply pattern matching, substitute variables
**Example**: (x·y)·z = x·(y·z) ⇒ (a·b)·c = a·(b·c) [same equation, different vars]

### Strategy 3: Algebraic Manipulation
**When**: E₁ provides rewrite rule for E₂
**Action**: Use E₁ to simplify E₂, check equality
**Example**: If E₁ is x·x = x, use it to rewrite x·x in E₂

### Strategy 4: Counterexample Search
**When**: Structural properties suggest FALSE
**Action**: Try small magmas (2-4 elements) systematically
**Priority**: Check red flag properties first

```
START: Evaluate E₁ ⇒ E₂

│  └─ Check if same identity element → Use proof (Strategy 3)

├─ E₂ commutativity?

```

## HUB EQUATIONS

Highly connected equations that imply many others:

### Identity Hub
- `e·x = x` (left identity)
- `x·e = x` (right identity)
- **Implications**: All unital properties, cancellation laws

### Associativity Hub
- `(x·y)·z = x·(y·z)`
- **Implications**: Power laws, nested parenthesis properties

### Idempotent Hub
- `x·x = x`
- **Implications**: Projection properties, fixed points

## SMALL MAGMA SEARCH ORDER

When searching for counterexamples, try in this order:

1. **Size 2** (4 possible operations)
   - Fastest to check
   - Many counterexamples exist here

2. **Size 3** (19683 possible operations)
   - Check symmetries to reduce space
   - Focus on red flag patterns

3. **Size 4** (4^16 possible operations - too many!)
   - Use property-based filtering
   - Only if no counterexample in size 2-3

**Optimization**: If E₁ requires a property that fails in size 2,
you already have a counterexample!
