## QUICK REFERENCE: Red Flag Patterns

When evaluating E₁ ⇒ E₂, watch for these structural red flags:

| Red Flag | Indicates | Confidence |
|----------|-----------|------------|
| Non-commutative operation | E₁ ⇒ commutativity likely FALSE | 95% |
| Non-associative operation | E₁ ⇒ associativity likely FALSE | 87% |
| No identity element | Identity implications likely FALSE | 65% |
| Mixed operation orders | Structural implications likely FALSE | 73% |

**Key insight**: If the magma shows ANY red flag for the target property,
the implication is probably false. Look for counterexamples in 2-4 element magmas.

## CRITICAL IMPLICATION RULES

### Identity Element Rules (NEW - v2)

**Rule 1: Two-Sided Identity ⇒ Unilateral Identity**
- IF E₁: "e*x = x AND x*e = x for all x" (both sides)
- AND E₂: "e*x = x" OR "x*e = x" (one side)
- THEN: TRUE (100% confidence)
- Reasoning: Two-sided identity implies each one-sided version

**Rule 2: Unilateral Identity ⇏ Two-Sided Identity**
- IF E₁: Only "e*x = x" OR only "x*e = x"
- AND E₂: "e*x = x AND x*e = x"
- THEN: FALSE (85% confidence)
- Counterexample: Left-only identity structures exist

### Associativity Extension Rules (NEW - v2)

**Rule 3: Standard Associativity ⇒ Extended Forms**
- IF E₁: "(x*y)*z = x*(y*z)" (standard 3-variable)
- AND E₂: Extended associativity with more variables
- THEN: Usually TRUE (75% confidence)
- Examples: "(x*y)*(z*w) = ((x*y)*z)*w" extends standard pattern

**Rule 4: Associativity ⇏ Commutativity**
- IF E₁: Any associativity pattern
- AND E₂: "x*y = y*x"
- THEN: FALSE (95% confidence)
- Counterexample: Matrix multiplication (associative, not commutative)

### Reflexive/Symmetric Rules

**Rule 5: Reflexive Implication**
- IF E₁ and E₂ are identical (possibly renamed variables)
- THEN: TRUE (100% confidence)
- Example: "x*y = y*x" ⇒ "a*b = b*a" (same structure)

## EQUATION TAXONOMY

### Structural Classes

**Associative**: ((x·y)·z) = (x·(y·z))
- Creates semigroup structure
- Enables parentheses omission
- Common in: addition, multiplication
- Extends to: larger nested expressions

**Commutative**: (x·y) = (y·x)
- Order-independent operations
- Common in: addition, AND, OR, XOR
- Does NOT imply associativity

**Idempotent**: (x·x) = x
- Elements are projections
- Common in: meet, join, min, max
- Does NOT imply commutativity

### Identity Properties

**Left Identity**: ∃e: (e·x) = x for all x
**Right Identity**: ∃e: (x·e) = x for all x
**Two-sided Identity**: Both hold (then e is unique)

**Compound Identity**: "e·x = x AND x·e = x"
- This is the full definition of identity element
- Implies both left and right versions
- Common cheat: check for "AND" in equation

### Distributive Properties

**Left Distributive**: x·(y·z) = (x·y)·(x·z)
**Right Distributive**: (x·y)·z = (x·z)·(y·z)

**Medial**: (x·y)·(z·w) = (x·z)·(y·w)
- Stronger than distributive in semigroups
- Implies certain symmetric properties

## PROOF STRATEGY GUIDE

### Strategy 1: Trivial/Reflexive Check
**When**: E₁ and E₂ have identical form (possibly renamed variables)
**Action**: Return TRUE
**Confidence**: 100%
**Example**: (x·y)·z = (x·y)·z ⇒ anything about itself
**Example**: x·y = y·x ⇒ a·b = b·a (same structure, different vars)

### Strategy 2: Identity Compound Check (NEW - v2)
**When**: E₁ contains "AND" or compound conditions
**Action**: Split and analyze each component
**Example**: "e·x = x AND x·e = x" means full identity
**Confidence**: 85-100%

### Strategy 3: Structural Similarity
**When**: E₁ and E₂ have same structure, different variables
**Action**: Apply pattern matching, substitute variables
**Example**: (x·y)·z = x·(y·z) ⇒ (a·b)·c = a·(b·c)

### Strategy 4: Algebraic Manipulation
**When**: E₁ provides rewrite rule for E₂
**Action**: Use E₁ to simplify E₂, check equality
**Example**: If E₁ is x·x = x, use it to rewrite x·x in E₂

### Strategy 5: Counterexample Search
**When**: Structural properties suggest FALSE
**Action**: Try small magmas (2-4 elements) systematically
**Priority**: Check red flag properties first

## DECISION TREE FOR IMPLICATIONS

```
START: Evaluate E₁ ⇒ E₂
│
├─ E₁ and E₂ identical (possibly renamed)? → TRUE (Strategy 1)
│
├─ E₁ contains compound identity ("AND")?
│  └─ E₂ about unilateral identity? → TRUE (Strategy 2)
│
├─ E₁ is associativity, E₂ is extended associativity? → TRUE (75%)
│
├─ E₁ provides identity, E₂ about identity?
│  ├─ E₁: two-sided, E₂: one-sided → TRUE
│  └─ E₁: one-sided, E₂: two-sided → FALSE
│
├─ E₂ commutativity?
│  ├─ Is E₁ "non-commutative" red flag? → FALSE
│  └─ Is E₁ "strongly commutative"? → Check structure
│
├─ E₂ associativity?
│  ├─ Is E₁ "non-associative" red flag? → FALSE
│  └─ Is E₁ "strongly associative"? → Check structure
│
└─ Default → Search counterexamples in 2-4 element magmas
```
